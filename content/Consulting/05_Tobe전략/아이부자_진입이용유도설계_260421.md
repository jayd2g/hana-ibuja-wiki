**작성일**: 2026-04-21
**작성자**: Claude (Cowork) / VINYLC
**대상**: 부모 트랙 / 자녀 트랙 — 권한 이양 5단계 × 조건·이벤트 축
**목적**: 각 단계·조건에서 "누가, 언제, 왜, 어떻게" 앱을 여는지 구체화. 검토된 기존 아이디어(배당 알림·기회비용 시각화·포트폴리오 공개·제안권·증여 정보)를 실제 진입·이용 시나리오로 재배치.

## 0. 설계 원칙

1. **역할 부재 해결이 근본, 유도는 외피**. 유도 자체가 목적이 아니라 역할(관찰자 → 독립 견습)을 부여하기 위한 진입점 설계이다.
2. **부모·자녀 비대칭**. 부모는 '계속 열 이유', 자녀는 '열어도 괜찮은 이유'가 다르다. 같은 메시지·같은 방법으로는 양쪽 모두 실패한다.
3. **연령 축 + 조건 축 병행**. 단계 내 정적 설계만으로는 리텐션 한계 → 이벤트 기반 트리거를 반드시 병행.
4. **감시감 최소화**. 중학생 이상 이탈의 일차 원인은 '감시앱 낙인'. 프라이버시 기본값·호출 화법이 유도보다 우선한다.
5. **투자권유 회피**. 유도 메시지는 '정보 제공·경험 설계'의 언어로 구성. 상품 특정 추천 금지(PART 8 문제 3 원칙).

### 2-1. 1단계 — 관찰자 (자녀 7~9세)

#### 부모 트랙 (최초 진입 시기)
| 항목 | 내용 |

| 진입 채널 | 하나원큐 "자녀 금융 습관" 배너 · 주거래 고객 대상 카카오/SMS · 어린이날·입학 시즌 캠페인 |
| 핵심 메시지 | "돈을 '경험'할 시기입니다. 부모의 손 안에서 안전하게 시작하세요." |
| 심리 지렛대 | 불안(늦기 전에) + 통제감(내 안에서) + 정체성(부자 가정의 금융 습관) |
| 첫 혜택 | 매칭 예치금 3만원 / 첫 달 매칭 2배 / 캐릭터 저금통 꾸미기 |
| 앱을 여는 반복 이유 | 주 1회 "자녀 용돈 배달" 푸시 · 주간 가족 돈 이야기 카드(3분) |

| 방법 | 캐릭터 저금통 시각화 · 스티커·색 선택 · 부모와 함께 여는 주간 카드 |
| 메시지 | "이번 주 용돈 3,000원이 도착했어요!" / "저금통이 어젯밤 조금 자랐어요" |
| 원칙 | 금액·% 대신 시각 성장. 선택 없음(관찰자 단계), 감상만. |
| 리스크 | 소비 상세 노출 X — 이 나이에는 '기록'이 아닌 '경험' |

### 2-3. 3단계 — 제안자 (13~15세) **— 중학교 이탈 핵심 구간**

#### 부모 트랙
| 항목 | 내용 |

| 반복 이유 | 월 1회 '가족 투자 회의' 템플릿 · 부모 포트폴리오 일부 공개(내 주거래 내역 일부 = 교육 소재) |

|------|------|

| 방법 | 제안 투표 기능(매칭금 일부 자녀 제안) · 배당·이자 푸시("자는 동안 네 돈이 일했어") · 포트폴리오 월간 카드 |
| 메시지 | "네 한 표가 이번 달 매칭금의 30%를 움직여." / "어젯밤 네 돈이 1,300원 벌었어." / "이번 뉴스, 네 포트에 이런 영향 있어." |
| 감시감 제거 | 소비 세부 부모 노출 **기본 OFF** · '사적 영역' UI 탭 분리 · 제안·적립은 공유, 소비는 사적 |

| 리스크 | 투자권유 해당 위험 — 자녀 제안 → 부모 승인의 **구조적 주체**는 부모. 앱은 정보·도구만 제공 |

## 4. "부모가 앱을 계속 여는 이유" — 미해결 질문 대응

> 프로젝트 핵심 미해결 질문: *"부모가 이 앱을 계속 열게 만드는 이유가 무엇인가?"*

다섯 가지 반복 트리거를 축으로 설계합니다.

1. **내 아이 전용 인사이트 피드** — 자녀 연령·성향 기반 금융 육아 콘텐츠(개인화). 일반 육아앱 대비 차별점은 "금융 교육에 직접 연동됨".
2. **가족 자산 뷰** — 나 + 배우자 + 자녀 통합. 하나은행 주거래·예금·대출 연동으로만 가능. 타 핀테크가 모방 불가한 영역.
3. **분기별 자녀 재무 성장 리포트** — 부모의 육아 성취감 자극. 체중·키 그래프의 금융 버전.
4. **부모 자신의 세무·증여 전략 도구** — 자녀 앱이지만 부모의 실익. 증여 시뮬레이터·세무사 연결·프리미어 연계.
5. **자녀가 올린 성과 알림** — 자녀 제안·절약·투표 이벤트가 부모를 앱으로 끌어오는 **관계 기반 트리거**.

이 중 **1+4**가 부모의 '자기 효용', **2+3+5**가 '가족 관계 효용'. 두 축을 병행해야 이탈 방어 가능.

## 5. "자녀가 계속 쓰는 이유" — 중학교 이탈 특별 대응

중학교 이탈은 자녀(감시앱 인식·또래 압박) + 부모(모니터링 니즈 감소) 양방향 문제. 자녀 측은 아래 다섯 장치로 방어합니다.

1. **감시감 제거** — 소비 세부는 **자녀만 보는 사적 영역**으로 분리. 부모 노출 기본 OFF. UI에 '사적' 탭 명시.
2. **또래 대신 가족** — 비교 프레임을 또래 랭킹에서 가족 누적치로 재설계. 또래 노출 0.
3. **기여감** — 제안·투표·배당 알림을 통해 "내가 이 돈을 키웠다" 서사를 자녀에게 부여. 단순 소비관리 앱과의 결정적 차이.
4. **현실 효용** — 배당·이자·매칭이 **실제로 쓸 수 있는 돈**으로 돌아옴. 추상적 저축이 아닌 즉시 체감되는 가치.
5. **소셜 방어** — '감시앱 낙인' 방지 UX 언어. '부모 승인' 대신 '가족 결정', '모니터링' 대신 '공유' 등 화법 설계.
